# 1.答案：不等于 #
## 原因：因为发生了两次精度丢失，二进制浮点数的小数位数只能存储52位，导致后面第53位为1则进位，为0则舍去 ##
## 在js中，都是以二进制进行存储和运算的 ##
## 第一次在0.1和0.2分别转换为二进制的时候，有一次精度丢失 ##
## 在第二次0.1+0.2的得数的时候，导致小数位多一位，又造成了一次精度丢失 ##
## 最终导致不等于0.3 ##
